Issue Description
The icons within the div:

html
Copy code
<div class="absolute top-4 right-4 flex gap-2">
  <button class="..."><svg ... style="color: rgb(255, 255, 255);"></svg></button>
  <button class="..."><svg ... style="color: rgb(255, 255, 255);"></svg></button>
</div>
are defaulting to white (rgb(255, 255, 255)) in light mode. Since the button backgrounds are also white, the icons are not visible. However, in dark mode, the icons display correctly.

Root Cause
CSS Style Conflict:

The style="color: rgb(255, 255, 255);" inline styles applied to the SVG elements force the icons to remain white regardless of the theme.
Theme-Specific Foreground Colors:

The button and icon colors are not being dynamically adjusted based on the theme (light or dark mode).
Hardcoding:

The icon color is hardcoded instead of being responsive to the Tailwind theme classes.
Proposed Solution
Remove Inline Styles:

Eliminate the style="color: rgb(255, 255, 255);" inline style from the SVG elements.
Use Tailwind's Theme Utilities:

Apply dynamic classes to the icons that respect the text-secondary-foreground utility, which adjusts the text/icon color based on the theme.
Dynamic Text Color:

Ensure the icon color is tied to the theme context, e.g., text-secondary-foreground for the foreground color in buttons.
Steps to Fix
Update the Component Code: Ensure your component dynamically applies text color using Tailwind's theme utilities. Example:

html
Copy code
<div class="absolute top-4 right-4 flex gap-2">
  <button class="bg-secondary text-secondary-foreground hover:bg-secondary/80 h-10 w-10 opacity-0 group-hover:opacity-100 transition-opacity rounded-[8px]">
    <svg xmlns="http://www.w3.org/2000/svg" class="h-4 w-4">
      <rect width="14" height="14" x="8" y="8" rx="2" ry="2"></rect>
      <path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"></path>
    </svg>
  </button>
  <button class="bg-secondary text-secondary-foreground hover:bg-secondary/80 h-10 w-10 opacity-0 group-hover:opacity-100 transition-opacity rounded-[8px]">
    <svg xmlns="http://www.w3.org/2000/svg" class="h-4 w-4">
      <rect width="18" height="11" x="3" y="11" rx="2" ry="2"></rect>
      <path d="M7 11V7a5 5 0 0 1 9.9-1"></path>
    </svg>
  </button>
</div>
Verify Tailwind Configuration:

Ensure text-secondary-foreground and bg-secondary are correctly defined in your tailwind.config.js file for both light and dark modes.
If these utilities are missing, extend your theme to include them.
Check Theme Context:

Verify that the app dynamically applies the light or dark class to the <html> or <body> tag when switching themes.
Test Across Themes:

Test the updated buttons in both light and dark modes to ensure the icons adapt correctly.
Expected Outcome
Light Mode: The icons appear dark and visible against white buttons.
Dark Mode: The icons remain white and visible against dark buttons.
The solution uses Tailwind utilities, ensuring maintainability and consistency with other components.
