Updated Feature Request: Undo/Redo Functionality with Keyboard Shortcuts
Objective
Enable users to undo and redo changes to their color palette using keyboard shortcuts. This feature should work seamlessly with the lock/unlock functionality and be available to both logged-in and non-logged-in users. Additionally:

If a user logs in after making changes, their complete history of changes before logging in should still be accessible.
The history should reset upon page refresh.
Key Requirements
Undo/Redo Keyboard Shortcuts:

Windows:
Undo: Ctrl + Z
Redo: Ctrl + Y
Mac:
Undo: Command + Z
Redo: Command + Shift + Z
Session-Limited History:

History is maintained in the browser session and reset upon page refresh.
Available to All Users:

The undo/redo feature should work for both logged-in and non-logged-in users.
Users who log in after making changes can still access their complete history.
Seamless Integration with Lock/Unlock:

Locked colors remain unaffected during undo/redo actions.
Only unlocked colors are changed.
No UI Buttons:

Undo/redo is triggered solely through the specified keyboard shortcuts.
Action Plan
1. Maintain State History
Create two stacks:
undoStack: Stores the history of palette states.
redoStack: Stores states that have been undone for potential redo.
Implementation Example:

tsx
Copy code
const [undoStack, setUndoStack] = useState<string[][]>([]);
const [redoStack, setRedoStack] = useState<string[][]>([]);
2. Update History on Changes
Push the current palette state to the undoStack whenever the user makes a change (e.g., generates a new palette or edits a color).
Clear the redoStack whenever a new change is made.
Example:

tsx
Copy code
const saveToHistory = useCallback(() => {
  setUndoStack(prev => [...prev, [...colors]]);
  setRedoStack([]); // Clear redo stack on new changes
}, [colors]);
3. Persist History Across Login
Ensure the undoStack and redoStack are not cleared when a user logs in.
The stacks should persist through the transition from logged-out to logged-in states.
Example:

tsx
Copy code
useEffect(() => {
  if (user) {
    console.log("[Undo/Redo] User logged in. Preserving history stacks.");
  }
}, [user]);
4. Implement Undo/Redo Functions
Undo:
On Ctrl + Z (Windows) or Command + Z (Mac), pop the last state from undoStack and apply it to the palette.
Push the current palette to the redoStack.
Example:

tsx
Copy code
const undo = useCallback(() => {
  if (undoStack.length === 0) return; // No history to undo
  const previousState = undoStack.pop();
  setRedoStack(prev => [...prev, [...colors]]);
  setColors(previousState || []);
}, [undoStack, colors]);
Redo:
On Ctrl + Y (Windows) or Command + Shift + Z (Mac), pop the last state from redoStack and apply it to the palette.
Push the current palette to the undoStack.
Example:

tsx
Copy code
const redo = useCallback(() => {
  if (redoStack.length === 0) return; // No history to redo
  const nextState = redoStack.pop();
  setUndoStack(prev => [...prev, [...colors]]);
  setColors(nextState || []);
}, [redoStack, colors]);
5. Handle Locked Colors
During undo/redo:
Only change unlocked colors.
Keep locked colors unchanged.
Example:

tsx
Copy code
const applyState = (newState: string[]) => {
  setColors(prevColors => 
    prevColors.map((color, index) => lockedColors[index] ? color : newState[index])
  );
};
6. Add Keyboard Event Listeners
Listen for keydown events to detect keyboard shortcuts.
Ensure compatibility with both Windows and Mac.
Example:

tsx
Copy code
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    const isMac = navigator.platform.toUpperCase().indexOf('MAC') >= 0;
    if (isMac) {
      if (e.metaKey && e.key === 'z' && !e.shiftKey) {
        e.preventDefault();
        undo();
      } else if (e.metaKey && e.key === 'z' && e.shiftKey) {
        e.preventDefault();
        redo();
      }
    } else {
      if (e.ctrlKey && e.key === 'z') {
        e.preventDefault();
        undo();
      } else if (e.ctrlKey && e.key === 'y') {
        e.preventDefault();
        redo();
      }
    }
  };

  window.addEventListener('keydown', handleKeyDown);
  return () => window.removeEventListener('keydown', handleKeyDown);
}, [undo, redo]);
7. Clear History on Page Refresh
Reset undoStack and redoStack when the page is refreshed.
Example:

tsx
Copy code
useEffect(() => {
  setUndoStack([]);
  setRedoStack([]);
}, []); // Run once on component mount
8. Test the Feature
Test all scenarios:
Undo/redo while logged out.
Undo/redo after logging in.
Ensure locked colors are unaffected.
Validate keyboard shortcuts on both Mac and Windows.
Confirm history resets on page refresh.
Expected Outcome
Users can undo/redo changes with keyboard shortcuts.
Locked colors remain unaffected.
History persists when transitioning between logged-out and logged-in states.
History resets upon page refresh.
The feature works seamlessly for both logged-in and non-logged-in users.
